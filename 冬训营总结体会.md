# 三、寒假训练营遇到的问题以及解决方法

- 问题：Anaconda 安装后出现 PyCharm 冲突或“出现两个 PyCharm”
    - 解决：区分系统安装的 PyCharm 与 Anaconda 附带的；统一在项目层面创建独立虚拟环境（conda env / venv），在 PyCharm 中选择对应解释器

- 问题：Obsidian + Git 自动提交和拉取的参数调配复杂
    - 解决：我结合ai和网上的教程，按注记调整 Obsidian Git 插件参数，并不断的比较各个说法和建议，最后成功自动上传自己的第一份文件

- 问题：一开始不知道如何处理缺失值并不是 NaN（例如 '?'、空字符串），而导致判断失败
    - 解决：先用 df.replace(to_replace='?', value=np.nan)，再用 df.dropna() 或 df.fillna()；针对列采用不同填充值（均值/中位数/众数/固定值）


- 问题：绘图中文乱码或显示异常
    - 解决：plt.rcParams['font.sans-serif'] = ['Microsoft YaHei']，plt.rcParams['axes.unicode_minus'] = False；保存图像应在 plt.savefig(...) 先于 plt.show()。

- 问题：训练出现过拟合（训练集表现优、验证集差），一开始不知道如何解决，只是纸上谈兵
    - 解决：可以引入正则化函数、Dropout或者早停、增大训练集或降低模型复杂度

- 问题：因为缺乏实践经验，所以不知道如何调超参数
    - 解决：结合ai，了解相关问题的常用解决方法：先用粗粒度搜索+ 验证集曲线观察，再用学习率衰减、warmup 或学习率搜索工具（初步了解到的方法）


# 四、在项目中解决问题的例子

- 案例1：读写 CSV/TSV 时分隔符与编码导致数据错列或乱码
    - 处理：了解到有sep参数，可以通过明确 sep 参数，指定 encodin），用 pandas.read_csv来读取数据

- 案例2：合并两个 DataFrame 时索引不对导致大量 NaN（使用 concat/merge 选择错误）
    - 处理：检查合并键（on 参数）是否存在重复或类型不一致（字符串 vs 数字），必要时统一类型再 merge；若只想垂直堆叠，使用 pd.concat(..., axis=0)

- 案例3：图像保存失败或中文无法显示
    - 处理：一开始是我将savefig和show方法的顺序搞混，所以要先设置 rcParams 支持中文并在 plt.savefig(path, dpi=...) 前调用，确保文件路径存在且权限可写；show 后 figure 释放导致 save 无效，要注意顺序：savefig -> show
